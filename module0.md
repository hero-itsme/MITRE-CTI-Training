# Module 0: Introducing MITRE ATT&CK® for CTI

This module serves as the foundation and roadmap for the full training series, establishing the core definitions and utility of the framework in threat intelligence.

---

## 🎯 1. Core Definition: Cyber Threat Intelligence (CTI)
The training adopts Sergio Caltagirone’s definition of CTI[cite: 1]:
* **Definition:** Actionable knowledge about adversaries and their activities[cite: 1].
* **Goal:** Enables defenders to make better security decisions and reduce harm[cite: 1].

## 🏗️ 2. How ATT&CK Enhances CTI
The primary value of the ATT&CK framework is its ability to **structure data**[cite: 1]. This allows analysts to:

### **Compare Behaviors**
* Contrast different threat groups[cite: 1].
* Track how a single group evolves over time[cite: 1].
* Compare adversary tactics against existing internal defenses[cite: 1].

### **Provide a Common Language**
* **Analyst to Defender:** Identifying a specific technique (e.g., *T1547.001 - Registry Run Keys*) allows a defender to immediately know which data sources (Registry data) are needed for detection[cite: 1].
* **Community Sharing:** Organizations can share intelligence using standardized IDs that are universally understood[cite: 1].

## 🚀 3. Training Goals
The course outlines five key objectives[cite: 1]:
1. **Understand Utility:** Why ATT&CK is the standard for CTI[cite: 1].
2. **Mapping:** How to extract ATT&CK techniques from narrative reports and raw data[cite: 1].
3. **Data Management:** How to store and visualize mapped data[cite: 1].
4. **Analysis:** How to perform threat analysis using the mapped data[cite: 1].
5. **Actionability:** How to turn intelligence into concrete defensive recommendations[cite: 1].

## 🗺️ 4. Roadmap of the Training Series
Module 0 sets the stage for the following lessons[cite: 1]:
* **Module 1:** Mapping from Narrative Reporting (External reports)[cite: 1].
* **Module 2:** Mapping from Raw Data[cite: 1].
* **Module 3:** Storing and Analyzing (Using tools like ATT&CK Navigator)[cite: 1].
* **Module 4:** Making Defensive Recommendations[cite: 1].
